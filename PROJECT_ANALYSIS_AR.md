# دراسة تحليلية شاملة لنظام المحاسبة وإدارة المعهد (Accounting System)

## 1. مقدمة
هذا المشروع هو نظام متكامل لإدارة معهد تعليمي (معهد الخنساء) برمجياً ومحاسبياً. تم تطويره باستخدام لغة **Visual Basic .NET**، ويستهدف أتمتة العمليات الإدارية والمالية، بدءاً من تسجيل الطالبات وصولاً إلى إدارة رواتب الموظفين والمصروفات العامة.

## 2. الهيكل التقني للنظام
*   **لغة البرمجة:** Visual Basic .NET.
*   **إطار العمل:** .NET Framework 4.7.2.
*   **قاعدة البيانات:** Microsoft Access (`users.accdb`) عبر محرك OLEDB.
*   **واجهة المستخدم:** Windows Forms مع استخدام مكتبة **Guna UI** لتقديم واجهات عصرية وتفاعلية.
*   **التقارير والمخرجات:**
    *   **iTextSharp:** لتوليد تقارير بصيغة PDF.
    *   **Excel Interop:** لتصدير البيانات إلى ملفات Excel.
    *   **PrintDocument:** للطباعة المباشرة من النظام.

## 3. تحليل الوحدات الوظيفية

### أ. لوحة التحكم (Dashboard)
تعد الواجهة المركزية التي تعرض إحصائيات فورية حول:
*   إجمالي عدد الطالبات، الموظفين، والأقسام.
*   الإيرادات المالية الكلية (رسوم طلاب + دعم خارجي).
*   المصروفات الكلية وصافي الربح.
*   رسوم بيانية توضح توزيع الطالبات حسب الأقسام وأنواع الدعم المالي (باستخدام مكتبة Charting).

### ب. إدارة الطالبات والرسوم (Student Management)
*   **التسجيل:** إضافة طالبة جديدة مع ربطها بتخصص ومستوى دراسي محدد.
*   **المحاسبة الطلابية:** متابعة الرسوم الدراسية، المبالغ المدفوعة، والمتبقية.
*   **السندات:** توليد سندات قبض تلقائية عند دفع الرسوم.
*   **التقارير:** كشوفات تفصيلية بأسماء الطالبات وحالاتهن المالية.

### ج. إدارة الموظفين والرواتب (HR & Payroll)
*   تسجيل بيانات الموظفين والمسمى الوظيفي ونوع التعاقد.
*   حساب الرواتب الشهرية مع إضافة المكافآت أو خصم الغياب.
*   إصدار سندات صرف الرواتب وربطها بالمصروفات العامة.

### د. إدارة المصروفات (Expense Management)
*   تصنيف المصروفات (إيجار، كهرباء، صيانة، إلخ).
*   تسجيل العمليات المالية مع إمكانية البحث والتعديل والحذف باستخدام نمط **DAL (Data Access Layer)**.
*   تتبع المصروفات عبر تقارير دورية.

### هـ. الدعم والمساهمات (Support & Revenue)
*   إدارة التبرعات والدعم الخارجي من منظمات أو فاعلي خير.
*   تخصيص الجهات المستفيدة من الدعم (الطلاب، الموظفين، أو المعهد).

### و. الأمن والصلاحيات
*   نظام تسجيل دخول يعتمد على أدوار المستخدمين (**مدير** أو **سكرتارية**).
*   التحكم في تفعيل أو تعطيل الأزرار بناءً على الصلاحيات.
*   ميزة استعادة كلمة المرور عبر سؤال الأمان.
*   نظام النسخ الاحتياطي واستعادة قاعدة البيانات يدوياً.

## 4. تحليل قاعدة البيانات
النظام يعتمد على جداول مترابطة في Access تشمل:
*   `Students`: لبيانات الطالبات الشخصية.
*   `Employee`: لبيانات الموظفين.
*   `Expenses`: لتتبع المصروفات اليومية.
*   `payment`: لتتبع إيرادات الرسوم الدراسية.
*   `ReceiptVoucher` & `ExpenseVouchers`: لتوثيق السندات المالية الصادرة والواردة.
*   `Users`: لإدارة الحسابات والصلاحيات وكلمات المرور.

## 5. تقييم النظام والمميزات
1.  **الشمولية:** يغطي كافة الجوانب الإدارية والمالية للمعهد في مكان واحد.
2.  **واجهة المستخدم:** جذابة وسهلة الاستخدام بفضل Guna UI والسمات (Themes).
3.  **تعدد صيغ التقارير:** دعم PDF و Excel والطباعة المباشرة يجعله مرناً في التعامل مع البيانات.
4.  **التكامل:** الربط التلقائي بين العمليات (مثل ربط دفع الرسوم بتوليد السند وتحديث الإحصائيات).

## 6. التوصيات التطويرية
1.  **الترقية إلى SQL Server:** لزيادة الأداء والأمان ودعم عدد أكبر من المستخدمين المتزامنين (Multi-user support).
2.  **تشفير البيانات:** تشفير كلمات المرور في قاعدة البيانات (حيث أنها مخزنة كنص واضح حالياً).
3.  **التحقق من البيانات:** تحسين آليات التحقق من المدخلات (Data Validation) لمنع الأخطاء البشرية (مثل منع إدخال نصوص في حقول الأرقام).
4.  **النظام السحابي:** التفكير في تحويل النظام إلى تطبيق ويب أو ربطه بقاعدة بيانات سحابية للوصول من أي مكان.
